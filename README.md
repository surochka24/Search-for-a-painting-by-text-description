# Поиск картин по текстовому описанию

## Описание проекта

**Проблема:** 
традиционный поиск изображений обычно основан на метаданных: названии картины, имени художника, жанре или других характеристиках. Однако на практике пользователь часто не знает этой информации и может описать изображение только словами, например: «зимний пейзаж с деревьями», «портрет женщины в красном платье» или «абстрактная картина с яркими цветами». Проблема заключается в том, что компьютер должен понять смысл текстового описания и сопоставить его с содержимым изображений. Для этого необходимо представить как текст, так и изображения в едином пространстве признаков, где похожие объекты будут располагаться близко друг к другу. Современные мультимодальные модели позволяют решать эту задачу без ручной разметки признаков и сложных правил сопоставления. Они автоматически извлекают смысловую информацию из текста и изображений и позволяют находить наиболее подходящие результаты по смыслу запроса.

**Цель проекта:** разработать систему поиска произведений искусства по текстовому описанию. Пользователь вводит текстовый запрос (например, "sunset over the sea" или "portrait of a woman"), после чего система находит наиболее подходящие картины из коллекции WikiArt.

**Технологии:**
основой проекта является модель CLIP, разработанная компанией OpenAI. CLIP обучается на больших наборах пар «изображение - текст» и преобразует как изображения, так и текстовые описания в числовые векторы фиксированной длины, называемые эмбеддингами. В проекте изображения из датасета WikiArt сначала преобразуются в эмбеддинги и сохраняются. Затем пользовательский запрос также преобразуется в эмбеддинг. После этого между текстовым и графическими представлениями вычисляется косинусное сходство (cosine similarity). Чем выше значение сходства, тем ближе изображение к смыслу текстового запроса. Для исследования планируется сравнить две модели:

CLIP ViT-B/32 - базовая англоязычная версия CLIP;
Multilingual CLIP - мультиязычная версия, поддерживающая русский и другие языки.

Сравнение позволит оценить, насколько качество поиска зависит от языка запроса и насколько мультиязычные модели лучше справляются с русскоязычными описаниями картин. Такое исследование делает проект не только прикладным, но и исследовательским и объединяет темы курса:

- Компьютерное зрение (Computer Vision);
- Трансформеры и мультимодальные нейронные сети;
- Векторные представления (Embeddings);
- Поиск по сходству (Similarity Search).

## Используемые данные

В проекте используется датасет WikiArt. Ссылка: https://huggingface.co/datasets/huggan/wikiart

- Полный размер датасета: более 80 000 произведений искусства.
- Для экспериментов была сформирована подвыборка из 10 000 изображений.

Для каждой картины доступны следующие метаданные:

- artist - художник;
- genre - жанр;
- style - художественный стиль.

---

## Используемые методы

### 1. Подготовка данных

Из датасета WikiArt была сформирована подвыборка из 10 000 изображений. Дополнительно была сохранена таблица метаданных:

- путь к изображению;
- художник;
- жанр;
- стиль.

---

### 2. Получение эмбеддингов изображений

Для преобразования изображений в векторные представления использовалась модель:

- openai/clip-vit-base-patch32

Каждая картина преобразуется в embedding фиксированной размерности. Полученные эмбеддинги сохраняются для последующего поиска.

---

### 3. Поиск по текстовому запросу

Текстовый запрос пользователя преобразуется в embedding той же моделью CLIP. Далее вычисляется косинусное сходство (Cosine Similarity) между текстовым embedding и эмбеддингами всех изображений. Система возвращает Top-K наиболее похожих картин.

---

### 4. Оценка качества

Для оценки качества поиска формируется набор тестовых запросов.

Примеры запросов:

- sunset over sea
- portrait of woman
- snowy landscape
- flowers in vase

Качество оценивается по тому, находится ли ожидаемая картина среди первых результатов поиска (Recall@K).

## Запуск проекта

### Шаг 1. Подготовка данных

```bash
01_prepare_dataset.ipynb
```

Формируется подвыборка WikiArt и таблица метаданных.

### Шаг 2. Вычисление эмбеддингов

```bash
02_compute_embeddings.ipynb
```

Для всех изображений вычисляются CLIP embeddings.

### Шаг 3. Поиск по тексту

```bash
03_search.ipynb
```

Введите текстовый запрос и получите наиболее похожие картины.

```bash
04_evaluation.ipynb
```

Запуск экспериментов и расчет метрик качества.

## Результаты

В результате была реализована система поиска произведений искусства по текстовому описанию.

Модель CLIP успешно сопоставляет текстовые запросы и изображения в общем пространстве признаков и позволяет находить визуально релевантные картины.

Примеры запросов:

- "sunset over sea"
- "portrait of woman"
- "snowy landscape"
